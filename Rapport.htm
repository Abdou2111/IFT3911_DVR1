1. Hypothèses

Nous avions comme hypothèse pour la réalisation de ce devoir les suivantes :

  • Chaque voyage a une heure de départ et d'arrivée. en plus les leux de départ et d,arrivée doivent etre de même type
  • Un lieu de transport peut contenir plusieurs compagnies
  • Une compagnie peut se situer dans plusieurs lieux de transport
  • La consultation retourne uniquement les sieges libres
  • les méthodes modifier prennent tous les parametres ( villes , pays) et le systeme traite les chaises vides
  • On a aussi assumé que tous les types de voyage ont des Ids
  • La disponibilité est recalculée à chaque recherche.

    3. Design Logiciel

    3.1 Diagramme de classe logiciel

    Le design logiciel applique les principes GRASP :

    • Expert : Les sections calculent la disponibilité.
    • Creator : Le contrôleur crée les réservations.
    • Faible couplage : Le système délègue aux contrôleurs.
    • Haute cohésion : Chaque classe a une responsabilité claire.
